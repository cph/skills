The Path to Mastery

Goals

I want to write a new Performance Review form specific to us. It should:

  1. Focus on our technical skills
  2. Use an absolute scale rather than one relative to our current titles
  3. Make achievements transparent
  4. Be the product of an open process

Example

A review could look like this:

Ruby
Javascript/Coffeescript
Git
jQuery/DOM
OO Programming

Explanation

My plan is to group our skills into four categories:

  1. Dexterity with Tools
  2. Fluency with Languages
  3. Knowledge of Frameworks
  4. Mastery of Techniques

And to describe five milestones for each:

  1. Novice
    • Unaware of context
    • Must follow context-free rules
  2. Advanced Beginner
    • Recognizes aspects of context
    • Can start applying context-sensitive rules
  3. Competent
    • Appreciates breadth of context; analyzes it slowly
    • Applies rules of thumb
  4. Proficient
    • Has memoized analysis of very many contexts
    • Makes decisions consciously
  5. Master
    • Acts instinctively
(Read more about the Dreyfus model in their original paper.)

I would like your help

  • Refining the concept
  • Finalizing the list of skills to include
  • Describing milestones for each skill
